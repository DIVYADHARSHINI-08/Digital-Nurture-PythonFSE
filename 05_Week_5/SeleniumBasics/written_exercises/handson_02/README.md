# Hands-On 2: Software Development Life Cycle (SDLC) and Testing Life Cycle (STLC)

## Objective

The objective of this hands-on exercise is to understand the Software Development Life Cycle (SDLC), Software Testing Life Cycle (STLC), Agile testing practices, and the role of Quality Assurance throughout the software development process.

---

## Topics Covered

- Software Development Life Cycle (SDLC)
- Software Testing Life Cycle (STLC)
- SDLC vs STLC
- V-Model
- Entry Criteria
- Exit Criteria
- QA Activities
- Agile Testing
- Shift-Left Testing
- Gherkin Scenarios

---

## Tasks Completed

- Compared SDLC and STLC.
- Studied the phases of the V-Model.
- Identified Entry and Exit Criteria for testing.
- Understood QA responsibilities during the development lifecycle.
- Learned Agile testing practices.
- Explored Shift-Left Testing concepts.
- Wrote sample Gherkin scenarios using Given-When-Then syntax.

---

## Learning Outcomes

After completing this hands-on, I was able to:

- Understand the relationship between SDLC and STLC.
- Explain the V-Model and its testing phases.
- Identify the conditions required to start and complete testing.
- Describe the responsibilities of QA engineers.
- Understand Agile testing methodology.
- Explain the benefits of Shift-Left Testing.
- Create basic BDD test scenarios using Gherkin.
