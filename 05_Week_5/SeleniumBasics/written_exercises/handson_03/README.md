# Hands-On 3: Test Automation Strategy and Framework Design

## Objective

The objective of this hands-on exercise is to understand test automation concepts, evaluate automation feasibility, compare automation frameworks, and design a suitable Selenium automation framework based on project requirements.

---

## Topics Covered

- Test Automation
- Automation Suitability Analysis
- Manual Testing vs Automation Testing
- Return on Investment (ROI)
- Flaky Tests
- Selenium Automation Frameworks
- Framework Selection
- Hybrid Framework Design

---

## Tasks Completed

- Identified test cases suitable for automation.
- Compared manual and automated testing approaches.
- Understood Return on Investment (ROI) in automation projects.
- Learned the causes and prevention of flaky tests.
- Compared different Selenium automation frameworks.
- Designed a Hybrid Selenium framework.
- Planned a scalable project folder structure.

---

## Learning Outcomes

After completing this hands-on, I was able to:

- Identify scenarios suitable for automation testing.
- Understand the benefits and limitations of automation.
- Explain ROI and its importance in automation projects.
- Recognize common causes of flaky tests and methods to improve test stability.
- Compare Data-Driven, Keyword-Driven, Modular, Linear, and Hybrid frameworks.
- Design a maintainable and reusable Selenium automation framework.
