# Hands-On 1: Software Testing Fundamentals

## Objective

The objective of this hands-on exercise is to understand the fundamental concepts of software testing, including testing types, defect management, and API testing. This exercise focuses on building a strong foundation in Quality Assurance (QA) concepts before moving to automation.

---

## Topics Covered

- Functional Testing
  - Unit Testing
  - Integration Testing
  - System Testing
  - User Acceptance Testing (UAT)

- Non-Functional Testing
  - Performance Testing

- Testing Techniques
  - Black Box Testing
  - White Box Testing

- REST API Testing
  - POST API Test Cases

- Defect Management
  - Defect Life Cycle
  - Severity vs Priority
  - Defect Report

---

## Tasks Completed

- Studied different software testing levels.
- Compared Black Box and White Box testing.
- Created sample REST API test cases.
- Understood the defect life cycle.
- Differentiated Severity and Priority.
- Prepared a sample defect report.

---

## Learning Outcomes

After completing this hands-on, I was able to:

- Understand the software testing process.
- Identify different testing levels.
- Differentiate functional and non-functional testing.
- Design basic API test cases.
- Understand how software defects are reported and tracked.
- Recognize the importance of QA in software development.

## Conclusion

This hands-on provided a strong understanding of software testing fundamentals and defect management practices. These concepts serve as the foundation for learning Selenium automation and building effective software quality assurance skills.
